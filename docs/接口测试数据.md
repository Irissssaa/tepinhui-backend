# 特品汇接口测试数据

> 本文档供测试人员直接复制接口请求进行测试。每个接口包含请求方法、路径、请求头、请求体（JSON）及预期响应。
>
> **测试前准备**：确保后端服务已启动（`./mvnw spring-boot:run`），测试数据已初始化（`db/data.sql`）。

## 认证登录

> 所有需要登录的接口，请在 Header 中携带 `Authorization: Bearer {token}`。

### 登录获取 Token

```
POST /tph/api/v1/auth/login
Content-Type: application/json

{
    "usernameOrPhone": "consumer1",
    "password": "test123"
}
```

**预期响应（200）：**
```json
{
    "code": 200,
    "message": "登录成功",
    "data": {
        "token": "eyJhbGciOiJIUzI1NiJ9...",
        "expiresIn": 86400
    }
}
```

### 测试账号一览

| 角色 | username | password | phone | 说明 |
|------|----------|----------|-------|------|
| ADMIN | admin | admin123 | 13800138000 | 平台管理员，可操作所有管理端接口 |
| CONSUMER | consumer1 | test123 | 13900139001 | 普通消费者 |
| CONSUMER | consumer2 | test123 | 13900139002 | 普通消费者 |
| MERCHANT | merchant1 | test123 | 13900139003 | 审核通过的商家（对应 merchant_id=3） |
| MERCHANT | merchant2 | test123 | 13900139004 | 审核通过的商家（对应 merchant_id=4） |

> 注：merchant1 对应 db/data.sql 中 user_id=3，merchant_id=3（user_id=4 → merchant_id=4）。

| 字段类型 | 格式 | 示例 |

| 注意 | T 是固定字符，连接日期和时间部分 | —— |

## 二、溯源模块 管理端 `/tph/api/v1/trace/admin`

> 需要管理员角色（ADMIN）。

### 2.1 待审核列表
```
GET /api/v1/trace/admin/pending?page=1&size=10
Header: Authorization: Bearer {admin_token}
```

### 2.2 审核详情
```
GET /api/v1/trace/admin/{id}
Header: Authorization: Bearer {admin_token}
```
**示例：** `GET /api/v1/trace/admin/2`

### 2.3 所有溯源列表
```
GET /api/v1/trace/admin/list?page=1&size=10
Header: Authorization: Bearer {admin_token}
```

### 2.4 审核溯源
```
PUT /api/v1/trace/admin/audit
Header: Authorization: Bearer {admin_token}
Content-Type: application/json

{
    "id": 1,
    "status": "pass",
    "remark": "信息核实无误，审核通过"
}
```

**status 可选值：** `pass`（通过）、`reject`（拒绝）

**审核通过响应（200）：**
```json
{"code": 200, "message": "溯源审核完成", "data": null}
```

**审核拒绝响应（200）：**
```json
{
    "id": 1,
    "status": "reject",
    "remark": "检测报告疑似造假，拒绝通过"
}
```

    "code": 200,

## 八、商家管理端 `/tph/api/v1/admin/merchant`

> 需要 ADMIN 角色。

### 8.1 待审核商家列表
```
GET /api/v1/admin/merchant/pending?page=1&size=10
Header: Authorization: Bearer {admin_token}
```

### 8.2 全部商家列表
```
GET /api/v1/admin/merchant/list?page=1&size=10
GET /api/v1/admin/merchant/list?page=1&size=10&status=approved
Header: Authorization: Bearer {admin_token}
```

### 8.3 商家详情
```
GET /api/v1/admin/merchant/{id}
Header: Authorization: Bearer {admin_token}
```
**示例：** `GET /api/v1/admin/merchant/5`

### 8.4 审核商家

**通过审核：**
```
PUT /api/v1/admin/merchant/{id}/audit
Header: Authorization: Bearer {admin_token}
Content-Type: application/json

{
    "status": "approved",
    "auditRemark": "资质齐全，审核通过"
}
```

**驳回申请：**
```
PUT /api/v1/admin/merchant/{id}/audit
Header: Authorization: Bearer {admin_token}
Content-Type: application/json

{
    "status": "rejected",
    "auditRemark": "营业执照信息与注册信息不符，请重新上传"
}
```

**status 可选值：** `approved`（通过）、`rejected`（驳回）

**审核通过响应（200）：**
```json
{"code": 200, "message": "商家审核完成", "data": null}
```

**非待审核状态（400）：**
```json
{"code": 400, "message": "该商家不是待审核状态"}
```

> **重要：审核通过后角色自动升级。** 后端会自动将该商家对应的 user 记录 role 从 CONSUMER 升级为 MERCHANT，无需手动操作。该用户需重新登录获取含 MERCHANT 角色的新 token 才能调用商家专属接口。

## 九、平台看板 `/tph/api/v1/admin/stats`

> 需要 ADMIN 角色。数据缓存在 Redis，TTL=30分钟。

```
GET /api/v1/admin/stats
Header: Authorization: Bearer {admin_token}
```

**预期响应（200）：**
```json
{
    "code": 200,
    "data": {
        "userCount": 25,
        "merchantCount": 3,
        "pendingMerchantCount": 1,
        "productCount": 12,
        "orderCount": 8,
        "totalSalesAmount": 2846.00,
        "traceCount": 5,
        "pendingTraceCount": 1
    }
}
```

**字段说明：**

| 字段 | 数据来源 | 说明 |
|------|----------|------|
| userCount | user 表 COUNT(*) | 全部注册用户 |
| merchantCount | merchant WHERE status='approved' | 审核通过的商家 |
| pendingMerchantCount | merchant WHERE status='pending' | 待审核商家 |
| productCount | product 表 COUNT(*) | 全部商品 |
| orderCount | orders 表 COUNT(*) | 全部订单 |
| totalSalesAmount | orders WHERE status IN (done,shipped) SUM(total_amount) | 已完成+已发货订单的销售总额 |
| traceCount | trace_record 表 COUNT(*) | 全部溯源记录 |
| pendingTraceCount | trace_record WHERE audit_status='pending' | 待审核溯源 |

**常见错误响应：**
```json
// 未登录
{"code": 401, "message": "未登录"}

// 非管理员
{"code": 403, "message": "禁止访问"}
```

## 十三、商品审核模块 `/tph/api/v1/admin/products`

> 仅 ADMIN 可操作。管理员审核商家提交的商品。

### 13.1 审核商品

> status=on 表示审核通过并上架，status=off 表示审核驳回或下架。

```
PUT /api/v1/admin/products/{id}/audit
Header: Authorization: Bearer {admin_token}
Content-Type: application/json

{
    "status": "on",
    "auditRemark": "商品信息完整，允许上架"
}
```

**status 可选值：** `on`（通过/上架）、`off`（驳回/下架）

**审核通过响应（200）：**
```json
{"code": 200, "message": "商品审核完成", "data": null}
```

**审核驳回响应（200）：**
```json
{
    "status": "off",
    "auditRemark": "商品描述与实物不符，下架处理"
}
```

**商品不存在（404）：**
```json
{"code": 404, "message": "商品不存在"}
```

**status 参数非法（400）：**
```json
{"code": 400, "message": "审核结果只能是 on 或 off"}
```
