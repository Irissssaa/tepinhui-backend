## 时间字段格式说明

| 字段类型 | 格式 | 示例 |
|---|---|---|
| `DATE`（日期）| `yyyy-MM-dd` | `2024-04-15` |
| `DATETIME`（日期时间）| `yyyy-MM-ddTHH:mm:ss` | `2024-04-15T10:30:00` |
| 注意 | T 是固定字符，连接日期和时间部分 | —— |

## 一、认证模块 `/tph/auth`

### 1.1 发送注册验证码
```
POST /auth/register/code
Content-Type: application/json

{
    "email": "test@example.com"
}
```

### 1.2 用户注册
```
POST /auth/register
Content-Type: application/json

{
    "username": "zhangsan",
    "password": "Aa123456!",
    "email": "test@example.com",
    "phone": "13800138000",
    "code": "123456"
}
```

### 1.3 用户登录
```
POST /auth/login
Content-Type: application/json

{
    "username": "zhangsan",
    "password": "Aa123456!"
}
```

**响应中的 token 字段，后续请求需在 Header 携带：**
```
Authorization: Bearer <token>
```

### 1.4 获取当前用户信息
```
GET /auth/me
Header: Authorization: Bearer <token>
```

### 1.5 刷新Token
```
POST /auth/refresh
Header: Authorization: Bearer <token>
```

### 1.6 用户登出
```
POST /auth/logout
Header: Authorization: Bearer <token>
```

## 二、商品模块 `/tph/api/v1/products`

> 需要登录（商家或管理员角色）。以下请求头示例使用商家token。

### 2.1 商品分页列表（公开）
```
GET /api/v1/products?page=1&size=10&keyword=龙井
```

### 2.2 商品详情（公开）
```
GET /api/v1/products/1
```

### 2.3 创建商品（商家）
```
POST /api/v1/products
Header: Authorization: Bearer <token>
Content-Type: application/json

{
    "merchantId": 1,
    "specialtyId": 1,
    "name": "西湖龙井特级",
    "description": "2024年明前西湖龙井，产自杭州狮峰山",
    "price": 399.00,
    "stock": 100,
    "images": [
        "https://example.com/img/longjing1.jpg",
        "https://example.com/img/longjing2.jpg"
    ],
    "status": "on"
}
```

### 2.4 更新商品（商家）
```
PUT /api/v1/products/1
Header: Authorization: Bearer <token>
Content-Type: application/json

{
    "name": "西湖龙井特级（2024新茶）",
    "description": "更新描述",
    "price": 359.00,
    "stock": 80,
    "images": ["https://example.com/img/longjing1_new.jpg"],

}
```

### 2.5 删除商品（商家/管理员）
```
DELETE /api/v1/products/1
Header: Authorization: Bearer <token>
```

## 三、溯源模块 C端 `/tph/api/v1/trace`

> 查询接口公开，录入接口需商家角色。

### 3.1 查询溯源（公开）
```
GET /api/v1/trace/TP-ZJ-LJ-2024-042138
```

### 3.2 商品溯源列表（公开）
```
GET /api/v1/trace/product/1
```

### 3.3 录入溯源（商家）
```
POST /api/v1/trace
Header: Authorization: Bearer <token>
Content-Type: application/json

{
    "productId": 1,
    "batchNo": "B20240415001",
    "originAddress": "浙江省杭州市西湖区龙井路88号",
    "longitude": 120.129870,
    "latitude": 30.251830,
    "produceDate": "2024-04-05",
    "producerName": "杭州狮峰龙井茶业有限公司",
    "rawMaterial": "鲜叶（龙井43号品种）",
    "processFactory": "杭州龙井茶厂",
    "processDate": "2024-04-06",
    "processDesc": "摊放→青锅→回潮→辉锅→整理，共6道工序",
    "inspectOrg": "浙江省茶叶质量检测中心",
    "inspectDate": "2024-04-08",
    "inspectResult": "合格",
    "inspectReportUrl": "https://example.com/reports/inspection2024.pdf",
    "warehouseInTime": "2024-04-09T08:00:00",
    "warehouseOutTime": "2024-04-10T14:30:00",
    "logisticsInfo": "从杭州仓出发，经顺丰速运至上海仓，配送至客户"
}
```

**溯源码格式：** `TP-{省份}-{品类}-{年份}-{6位随机数}`
**省份缩写参考：** 浙江→ZJ、江苏→JS、安徽→AH、福建→FJ、广东→GD、四川→SC 等
**品类缩写参考：** 龙井→LJ、碧螺春→BLC、普洱→PE、乌龙→WL、白茶→BC 等

### 3.4 审核溯源（管理员）
```
GET /api/v1/trace/admin/pending?page=1&size=10
```

### 3.5 审核溯源（管理员）
```
PUT /api/v1/trace/admin/audit
Header: Authorization: Bearer <token>
Content-Type: application/json

{
    "id": 1,
    "status": "pass",
    "remark": "信息核实无误，审核通过"
}
```

**status 可选值：** `pass`（通过）、`reject`（拒绝）
