## 三、溯源码规范（严格按 trace_solution.html）

### 3.1 溯源码命名规则

```
TP - {省份缩写} - {品类缩写} - {年份} - {6位随机数}
```

**示例：**
```
TP-ZJ-LJ-2024-042138
```

| 字段 | 含义 | 说明 |
|---|---|---|
| `TP` | 前缀 | 固定为 TP |
| `ZJ` | 省份缩写 | 浙江 |
| `LJ` | 品类缩写 | 龙井 |
| `2024` | 生产年份 | 四位年份 |
| `042138` | 随机批次号 | 6位数字，保证唯一性 |

## 四、溯源查询流程（严格按 trace_solution.html）

### 4.1 查询链路（6步）

```
1. 消费者触发查询（扫描二维码 / 手动输入溯源码）
        ↓
2. 后端优先查 Redis 缓存（key: trace:{traceCode}）
        ↓（未命中）
3. 查询 MySQL：trace_record WHERE trace_code = ? AND audit_status = 'pass'
        ↓
4. 联查 product、merchant 表，组装完整数据
        ↓
5. 写入 Redis 缓存（TTL=24h），返回前端
        ↓
6. 前端按六层结构渲染：产品 → 产地 → 生产 → 加工 → 质检 → 流通
```

### 4.2 缓存策略

| 缓存Key | 过期时间 | 说明 |
|---|---|---|
| `trace:{traceCode}` | 24小时 | 溯源查询结果缓存 |
| `trace:stats` | 1小时 | 溯源统计缓存 |
| `trace:list:*` | 10分钟 | 列表缓存（分页） |

```java
// 溯源记录
@TableName("trace_record")
public class TraceRecord {

    private String inspectResult;     // 检测结果
```

### 7.1 溯源码生成（张圯炫）

```java
private String generateTraceCode(Long productId, String originProvince) {
    // 1. 通过 productId 查询 specialty，再获取 category
    Product product = productMapper.selectById(productId);
    Specialty specialty = specialtyMapper.selectById(product.getSpecialtyId());
    String categoryAbbr = getCategoryAbbr(specialty.getCategory());

    // 2. 省份缩写
    String provinceAbbr = getProvinceAbbr(originProvince);

    // 3. 年份 + 随机数
    String year = LocalDate.now().getYear().toString();
    String random = String.format("%06d", new Random().nextInt(999999));

    // 4. 组装
    return String.format("TP-%s-%s-%s-%s", provinceAbbr, categoryAbbr, year, random);
}

private String getProvinceAbbr(String province) {
    Map<String, String> MAP = Map.ofEntries(
```

### 7.2 溯源查询 + 缓存（张圯炫）

```java
@Cacheable(value = "trace", key = "'trace:' + #traceCode", cacheManager = "cacheManager")
public TraceQueryVO getTraceInfo(String traceCode) {
    // 1. 查 MySQL
    TraceRecord record = traceMapper.selectByTraceCode(traceCode);
    if (record == null) {
        throw new BusinessException(404, "溯源信息不存在");
    }
    if (!"pass".equals(record.getAuditStatus())) {
        throw new BusinessException(404, "溯源信息未审核通过");
    }

    // 2. 联查 product、specialty、merchant
    Product product = productMapper.selectById(record.getProductId());
    Specialty specialty = specialtyMapper.selectById(product.getSpecialtyId());
    Merchant merchant = merchantMapper.selectById(product.getMerchantId());

    // 3. 组装 VO
    return buildTraceVO(record, product, specialty, merchant);
}
```

### 7.3 审核通过后生成二维码（张圯炫）

```java
public void auditTrace(TraceAuditDTO dto) {
    TraceRecord record = traceMapper.selectById(dto.getId());

    if ("pass".equals(dto.getStatus())) {
        // 审核通过，生成二维码 URL
        String qrUrl = generateQRCodeUrl(record.getTraceCode());
        record.setQrUrl(qrUrl);
    }

    record.setAuditStatus(dto.getStatus());
    record.setAuditRemark(dto.getRemark());
    traceMapper.updateById(record);

    // 清除缓存
    redisTemplate.delete("trace:" + record.getTraceCode());
}
```
